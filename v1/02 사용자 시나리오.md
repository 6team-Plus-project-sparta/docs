## 2. 핵심 사용자 시나리오

### 시나리오 1 — 콘서트 티켓 선착순 예매 (Happy Path)

**배경:** 인기 아이돌 콘서트 티켓이 오늘 오후 2시에 오픈된다. 지수는 점심시간부터 준비 중이다.

```
[오후 1:50] 지수가 TicketFlow에 접속해 로그인한다.
[오후 1:55] 검색창에 "아이유 콘서트"를 입력한다.
            → 인기 검색어 1위에 "아이유"가 이미 올라와 있다.
[오후 1:58] 이벤트 상세 페이지에서 구역 선택 화면을 확인한다.
[오후 2:00] 오픈과 동시에 A구역 15열 23번 좌석을 선택한다.
            → 서버: Redis 분산락 획득 → 재고 1 차감 → 임시 선점 (5분)
[오후 2:01] 보유 쿠폰 "신규가입 20% 할인"을 적용한다.
[오후 2:02] Mock PG 결제 완료 → 예매 확정
[오후 2:03] "예매 완료" 문자·이메일 수신 (시스템 메시지)
[오후 2:10] 마이페이지에서 예매 내역과 QR 티켓을 확인한다.
```

**기술 포인트:**
- 오후 2:00 순간 수천 명 동시 접속 → Redis 분산락으로 재고 정합성 보장
- 5분 내 미결제 시 선점 자동 해제 (TTL 기반)
- 검색 시 Caffeine 캐시로 DB 부하 없이 즉시 응답

### 시나리오 2 — 선착순 쿠폰 발급 실패 (Sad Path)

**배경:** 한정 100장의 "여름 특가 쿠폰" 오픈. 지수는 조금 늦었다.

```
[오후 3:00] 쿠폰 이벤트 페이지에 접속한다.
[오후 3:00:05] "쿠폰 받기" 버튼을 클릭한다.
            → 서버: Redis SETNX로 Lock 시도 → 재고 확인 → 이미 100/100
[오후 3:00:05] "죄송합니다. 쿠폰이 모두 소진되었습니다." 메시지 수신
```

**기술 포인트:**
- 100명 초과 발급 절대 불가 → AtomicLong 또는 Redis 원자적 DECR
- 동시에 101번째 요청이 와도 재고 음수 불가

### 시나리오 3 — CS 문의 채팅 (실시간 상태 관리)

**배경:** 지수가 예매한 공연 날짜가 변경되었다는 공지를 봤다.

```
[지수] 마이페이지 → "1:1 문의하기" 클릭
      → 채팅방 생성 (status: WAITING)
      → "공연 날짜 변경으로 환불 가능한가요?" 메시지 전송

[민준] CS 어드민 화면에서 새 문의 알림 수신
      → 문의 상태를 WAITING → IN_PROGRESS로 변경
      → "안녕하세요. 확인 후 안내드리겠습니다." 실시간 답변

[민준] 처리 완료 후 상태를 IN_PROGRESS → COMPLETED로 변경

[지수] 실시간으로 답변 수신, 문의 종료 확인
```

**기술 포인트:**
- WebSocket(STOMP) 양방향 실시간 통신
- 문의 상태 ENUM: `WAITING → IN_PROGRESS → COMPLETED`
- JWT 기반 STOMP CONNECT 시점 인증

### 시나리오 4 — 이벤트 등록 (어드민)

**배경:** 현우가 새 뮤지컬 공연 정보를 등록한다.

```
[현우] 어드민 로그인 (ROLE_ADMIN)
[현우] 이벤트 등록: 공연명, 장르, 날짜, 장소, 포스터 이미지
[현우] 구역 설정: A구역(VIP) 100석, B구역 200석, C구역 300석
[현우] 좌석별 가격 설정: A구역 15만원, B구역 10만원, C구역 7만원
[현우] 프로모션 쿠폰 등록: "얼리버드 10%" 100장 한정
[시스템] 이벤트 활성화 → 사용자 검색 노출 시작
```

### 시나리오 5 — 인기 검색어 동작

**배경:** 많은 사용자가 같은 키워드를 검색한다.

```
[10:00~11:00] "BTS", "아이유", "손흥민" 키워드 집중 검색
[시스템] 검색 시마다 Redis ZSet에 ZINCRBY keyword 1
[사용자] 메인 화면 인기 검색어 위젯 조회
       → Caffeine 캐시 HIT (TTL 5분)
       → Top 10 키워드 즉시 반환: ["BTS", "아이유", ...]
[5분 후] 캐시 만료 → 다음 요청 시 Redis ZSet에서 재집계 후 캐시 갱신
```

## 3. 사용자 여정 요약

```
일반 사용자 여정
  회원가입/로그인 → 이벤트 탐색(검색) → 이벤트 상세 → 좌석 선택
  → 쿠폰 적용 → 결제(Mock) → 예매 확정 → 마이페이지 조회
  → (필요 시) CS 문의 채팅

어드민 여정
  로그인 → 이벤트 등록 → 구역/좌석 설정 → 쿠폰 등록
  → 예매 현황 조회 → CS 문의 처리
```
