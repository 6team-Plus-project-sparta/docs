## 3. EC2 내부 Docker 구성

```mermaid
graph TB
    subgraph EC2["EC2 인스턴스"]
        subgraph DockerNetwork["Docker Network (ticketflow-net)"]
            SpringApp["🍃 Spring Boot Container\n:8080\nimage: ECR/ticketflow/backend:latest"]
        end
        Nginx["🔁 Nginx (optional)\nReverse Proxy :80/:443"]
    end

    Internet["인터넷"] --> Nginx
    Nginx --> SpringApp
    SpringApp -- "외부 연결" --> RDS_EXT["RDS (Private)"]
    SpringApp -- "외부 연결" --> Redis_EXT["ElastiCache (Private)"]
```

> **로컬 개발 환경** (docker-compose.yml)
> ```
> services:
>   app: Spring Boot (빌드된 jar)
>   mysql: MySQL 8.0
>   redis: Redis 7
> ```
> 로컬에서는 docker-compose로 MySQL + Redis를 함께 실행하므로 AWS 없이 개발 가능

## 5. 민감 정보 관리 흐름

```mermaid
sequenceDiagram
    participant Dev as 개발자
    participant GH as GitHub Secrets
    participant GA as GitHub Actions
    participant ECR as AWS ECR
    participant EC2 as EC2
    participant SSM as SSM Parameter Store

    Dev->>GH: AWS_KEY, ECR_URI 저장
    GA->>GH: Secrets 주입
    GA->>ECR: Docker Push (AWS 인증)
    GA->>EC2: SSM Send-Command
    EC2->>SSM: 런타임 시 Parameter 조회
    SSM-->>EC2: DB비밀번호, JWT_SECRET, REDIS_URL
    EC2->>ECR: Docker Pull (IAM Role)
```

**Parameter Store 키 구조:**
```
/ticketflow/prod/db/password
/ticketflow/prod/db/url
/ticketflow/prod/jwt/secret
/ticketflow/prod/redis/url
```

## 6. 로컬 개발 환경 구성 (docker-compose.yml 예시)

```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_DATABASE: ticketflow_db
      MYSQL_ROOT_PASSWORD: local_password
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    command: redis-server --appendonly yes

volumes:
  mysql_data:
```

## 7. 배포 환경별 설정 분리

| 환경 | DB | Redis | 설정 파일 |
|------|----|----|-----------|
| local | Docker MySQL | Docker Redis | `application-local.yml` |
| prod | AWS RDS | AWS ElastiCache | `application-prod.yml` + SSM |

Spring Profile 활성화: `SPRING_PROFILES_ACTIVE=prod` (EC2 컨테이너 환경변수)
